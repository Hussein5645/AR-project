# AR-project (Google Drive Version)

## التحول من AWS S3 إلى Google Drive
تم استبدال التخزين عبر S3 بالاستخدام المباشر لروابط ملفات عامة في مجلد Google Drive. لا حاجة لمفاتيح وصول أو إعداد حساب داخل التطبيق.

## الطريقة الحالية
1. ارفع ملفاتك (GLB للنماذج أو JPG/PNG للصور البانورامية) يدوياً إلى المجلد العام في Google Drive من حسابك.
2. احصل على رابط المشاركة للملف (مشاركة > انسخ الرابط).
3. افتح صفحة `upload.html` لاستخرج معرف الملف (File ID) من الرابط.
4. انسخ معرف الملف والصقه في الحقل داخل الصفحة الرئيسية (`index.html`). اختر نوع المحتوى ثم اضغط تحميل.
5. سيُنشأ رابط يحتوي على باراميترات (`?id=FILE_ID&type=model|panorama`) ويُولَّد QR Code له.

## ملاحظات
- لا يمكن الاستعراض التلقائي لمحتويات المجلد بدون مفتاح API (تم تجنب ذلك تحقيقاً لشرط "بدون تسجيل").
- معرف الملف هو ما بعد `/d/` في رابط المشاركة القياسي.
- الروابط تُحمَّل بنمط `https://drive.google.com/uc?export=download&id=FILE_ID`.
- للواقع المعزز يعمل فقط مع نماذج GLB المدعومة عبر `<model-viewer>`.

## حدود
- لا يوجد رفع مباشر من الموقع إلى Drive بسبب قيود المصادقة في واجهة Google بدون استخدام OAuth أو API Key.
- إذا رغبت مستقبلاً في رفع مباشر، ستحتاج إلى تهيئة واجهة خلفية وسيط أو استخدام خدمة Functions مع OAuth.

## تحسينات مستقبلية مقترحة
- إضافة ملف `manifest.json` في المستودع لحفظ وصف لكل معرف ملف (الاسم، النوع، وصف موجز) وتسهيل مشاركته.
- دعم glTF غير مضغوط عبر التحويل المسبق إلى GLB لتقليل زمن التحميل.
